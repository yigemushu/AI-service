# 浏览器插件 MVP

这个插件用于日常客服网页：在网页聊天页面选中客户消息后，可以通过插件弹窗、右键菜单或快捷键发送到 AI 客服助手。

## 安装方式

1. 打开 Chrome 或 Edge。
2. 进入扩展管理页：
   - Chrome: `chrome://extensions`
   - Edge: `edge://extensions`
3. 打开“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择本目录：

```text
integrations/browser-extension
```

## 配置

打开插件后填写：

- 网站地址：例如 `http://localhost:3000` 或 `http://43.139.225.55:3000`
- Webhook Token：必须和同一个网站环境里的 `INBOX_WEBHOOK_TOKEN` 一致
- 我的昵称：可选，用于排除自己的名字
- 平台、业务类型、客户昵称、客户文件夹
- 默认动作：`直接生成回复` 或 `发送到消息中心`

点击“保存配置”后，插件会自动把网站地址归一化为 origin。比如：

- `http://127.0.0.1:3000/settings` 会保存为 `http://127.0.0.1:3000`
- `http://43.139.225.55:3000/messages` 会保存为 `http://43.139.225.55:3000`

## 使用方式

### 弹窗模式

1. 在聊天网页选中客户消息。
2. 点击插件图标。
3. 点击“读取网页选中文本”。
4. 点击“直接生成回复”即可在插件内看到客户诉求、缺失信息、风险点和推荐回复。
5. 点击“复制回复”后，手动粘贴回聊天平台。

这个模式主要用于手动选中文本、生成草稿和复制回复；闲鱼闭环请优先使用下面的自动同步和“工作台发送回闲鱼”流程。

### 右键菜单

选中网页文字后，右键点击“发送到 AI 客服助手”。插件会按“默认动作”执行：

- 直接生成回复：调用 `{网站地址}/api/analyze`
- 发送到消息中心：调用 `{网站地址}/api/inbox`

### 快捷键

默认快捷键：`Ctrl+Shift+Y`

选中客户消息后按快捷键，插件会按“默认动作”执行。

### 闲鱼自动同步

1. 先在插件里导入连接码，或手动填写网站地址和 Webhook Token。
2. 导入连接码后，插件会自动开启“在闲鱼聊天页自动同步新买家消息到消息中心”和“允许工作台把确认后的回复同步回闲鱼”；如果是手动填写配置，请确认这两个开关已开启。
3. 在浏览器里打开闲鱼聊天页，并保持账号已登录、页面打开。
4. 买家新消息出现在页面后，插件会自动发送到网站的 `/api/inbox`，网站消息中心点击“同步外部消息”即可看到。

这个功能只读取当前浏览器页面能看到的消息，不会自动登录、不会绕过验证码、不会自动给买家发送回复。若闲鱼页面结构变化，可能需要更新 `content-script.js` 里的聊天气泡识别规则。

### 闲鱼手动同步兜底

如果自动同步没有抓到消息，可以在当前闲鱼聊天页打开插件，点击“同步当前页最新消息”。插件会从当前聊天页识别最近一条买家消息，并发送到网站的 `/api/inbox`。

这个按钮适合真实页面适配测试，也适合作为自动监听不稳定时的人工兜底。

### 闲鱼页面诊断

在真实闲鱼聊天页打开插件，点击“诊断当前闲鱼页面”。诊断会显示页面是否被识别为闲鱼、是否找到聊天输入框和发送按钮、客户名猜测、消息候选数量、消息选择器命中、输入框候选和按钮候选。

如果诊断失败或回填失败，点击“复制诊断结果”，把文本和页面截图一起保存。这样可以直接根据候选元素更新 `content-script.js`，不用靠猜页面结构。

### 工作台发送回闲鱼

1. 在网站消息中心选择一个闲鱼客户文件夹。
2. 确认或生成推荐回复后，点击“发送回闲鱼”。
3. 网站会创建一个待发送任务，插件会自动读取这个任务。
4. 插件找到对应闲鱼聊天页，把回复填入输入框。
5. 如果插件设置为“直接代我点击发送”，会继续点击闲鱼发送按钮；导入连接码后的默认动作仍是只填入输入框，代点击发送需要商家主动切换。

这个能力依赖原消息里的 `sourceUrl`，所以建议从插件自动同步或“同步当前页最新消息”进来的消息开始测试。闲鱼页面必须保持登录；如果聊天页没打开，插件会尝试打开原链接。

## Token

服务端必须配置：

```env
INBOX_WEBHOOK_TOKEN=一串随机Token
```

插件里的 Webhook Token 必须和当前网站环境一致。本地网站用本地 Token，云端网站用云端 Token。不要把真实 Token 提交到 GitHub。

## 当前边界

- 插件不会自动登录、不会绕过验证码、不会读取浏览器看不到的消息。
- 默认只把工作台回复填入闲鱼输入框；只有插件设置为“直接代我点击发送”时，才会尝试点击发送按钮。
- “保存到订单”仍在网站消息中心或订单页完成；插件会提供“打开消息中心 / 保存订单”入口。
- 不同聊天网站结构不同，客户昵称和消息气泡识别可能需要继续适配。

## 开发自检

修改插件内容脚本后，先运行：

```powershell
cd C:\Users\nieti\OneDrive\Desktop\codex\AI-service
npm.cmd run smoke:extension
```

这个检查会用模拟闲鱼聊天页验证：页面诊断、最新买家消息识别、回复回填、链接不一致防误发、代点击发送。
