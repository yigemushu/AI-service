# 浏览器插件 MVP

这个插件用于验证电脑端流程：在网页聊天页面选中客户消息，然后发送到网站 `/api/inbox`，再回到消息中心同步。

## 安装方式

1. 打开 Chrome 或 Edge。
2. 进入扩展管理页面：
   - Chrome：`chrome://extensions`
   - Edge：`edge://extensions`
3. 打开“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择本文件夹：

```text
integrations/browser-extension
```

## 使用方式

1. 打开闲鱼网页版或任意客户聊天网页。
2. 用鼠标选中客户发来的消息。
3. 点击浏览器右上角插件图标。
4. 确认网站地址，例如：

```text
http://localhost:3000
```

线上时改成你的正式网站地址。

5. 选择平台和业务类型。
6. 可选：填写“我的昵称”，插件会排除这个名字，避免把自己识别成客户。
7. 点击“读取网页选中文本”。
8. 插件会尝试从当前聊天页自动识别客户昵称，并填入“客户昵称”和“客户文件夹”。
9. 点击“发送到消息中心”。
10. 回到网站 `/messages`，点击“同步外部消息”。

如果客户昵称识别不准，可以在插件里手动改一次再发送。后续同一个客户文件夹的新消息会合并到消息中心的同一个文件夹。

## Token

如果服务器设置了：

```env
INBOX_WEBHOOK_TOKEN=
```

插件里也要填写同一个 token。不要把真实 token 提交到 GitHub。

## 当前限制

- 这是验证版，不会自动发送消息。
- 需要用户自己选中客户消息。
- 客户昵称会尽量从网页标题、聊天页顶部和选中消息附近识别；不同平台页面结构不同，识别不到时仍需手动填写。
- “我的昵称”用于排除自己的名字；“客户文件夹”用于把同一客户的多条消息归在一起。
- 只负责把消息送进网站，AI 分析和回复确认仍在网站内完成。
- 后续可以升级为自动识别聊天气泡和侧边栏建议回复。
